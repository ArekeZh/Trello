# Trello
KANBAN Task Manager. 
Trello Board — это полнофункциональное веб-приложение для управления задачами, реализующее канбан-логику в стиле Trello.

О проекте Цель: создать современное приложение для организации задач, с разделением досок, списков и карточек, а также поддержкой многопользовательской работы и drag & drop.

Технологии:

-Backend: Python, Django, Django REST Framework

-Frontend: JavaScript (vanilla), HTML, CSS

-База: PostgreSQL

-Версионирование: Git

-Вдохновение: популярные трекеры задач — Trello, Jira, Monday.

Ключевые особенности: -Регистрация/авторизация пользователей (REST API + Token аутентификация).

-CRUD-операции для досок, списков (колонок) и карточек.

-Drag & Drop: -Перемещение карточек между списками. -Перемещение самих списков между досками.

-Модальное управление: создание/изменение/удаление через pop-up окна.

-Цветовая маркировка списков и inline-редактирование названий.

-Реактивный frontend, без перезагрузки страницы.

-Детальное разграничение прав: только свои доски/списки/карточки.

-Полностью реализован паттерн Repository на backend-е: -Логика доступа и запросов к данным вынесена в отдельные repository-классы (не смешиваются с представлениями). -Код проще поддерживать, тестировать и расширять.

Backend:

-Модели: Board, List, Card, User (кастомная)

-Views: APIView, ViewSets для ресурсов

Репозитории: доступ к данным через отдельные классы (например, boards.py, lists.py, cards.py)

Сервисы: бизнес-логика (например, auth_service для регистрации и логина)

Frontend: -Отдельные JS-файлы для каждой логики: главная, доска, авторизация -Полная работа с API -SPA-подход в части интерфейса

Запуск проекта Установите зависимости:

pip install -r requirements.txt Настройте .env (SECRET_KEY, DB_NAME и прочее).

Проведите миграции:

python manage.py migrate

Запустите сервер:

python manage.py runserver

Особенности архитектуры: -Repository pattern — отделение слоя доступа к данным, что обеспечивает:

-Простую поддержку и масштабирование

-Лёгкую замену ORM/баз данных или работу с мемори-репозиториями (тесты)

-Модульность — сервисы отделены от views, всё структурировано по принципам clean-architecture.

-Разделение frontend/backend — чистый JS, минимизация сторонних библиотек.
